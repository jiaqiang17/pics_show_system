## ADDED Requirements

### Requirement: 批量标签维护
系统 SHALL 支持管理员在单次操作中为多张作品图片新增或移除标签，所有标签更改需记录操作者与时间戳。

#### Scenario: 成功批量添加标签
- **GIVEN** 管理员在作品列表中勾选多条记录并选择“批量标签”
- **WHEN** 提交包含多个标签 ID 的请求
- **THEN** 系统为每条作品关联这些标签，并写入操作日志
- **AND** 返回成功数量与受影响作品 ID 列表

#### Scenario: 标签校验失败
- **GIVEN** 请求中包含不存在或已停用的标签 ID
- **WHEN** 后端校验请求
- **THEN** 系统拒绝本次批量操作并返回具体标签错误信息
- **AND** 不对任何作品写入更改

### Requirement: 标签组合筛选
系统 SHALL 支持按照单个或多个标签、款式等条件组合筛选作品列表，并返回命中数量以便前端分页展示。

#### Scenario: 多标签筛选成功
- **GIVEN** 管理员在筛选面板选择两个标签
- **WHEN** 发起带标签数组的查询请求
- **THEN** 系统返回同时包含这两个标签的作品列表
- **AND** 响应中包含总记录数与分页信息

#### Scenario: 无匹配结果
- **GIVEN** 标签组合未命中任何作品
- **WHEN** 查询请求执行
- **THEN** 系统返回空列表
- **AND** 总记录数显示为 0，提示保持可复用查询条件

### Requirement: 前端展示同步
系统 SHALL 将标签筛选能力扩展至前端画廊页面，允许访客通过标签快速定位作品，且需使用与后台一致的筛选参数。

#### Scenario: 访客按标签浏览
- **GIVEN** 访客在画廊选择某标签
- **WHEN** 前端调用筛选接口
- **THEN** 页面展示匹配作品并更新筛选标签状态
- **AND** 若结果为空，显示“暂无匹配作品”提示
